1. MODEL SIZING

Chosen quantization: INT4

Formula:
weights_on_disk = params × bytes_per_weight

Using INT4:
bytes_per_weight = 0.5

13B model:
13,000,000,000 × 0.5 bytes = 6,500,000,000 bytes ≈ 6.1 GiB

70B model:
70,000,000,000 × 0.5 bytes = 35,000,000,000 bytes ≈ 32.6 GiB

Practical deployed VRAM note (supporting check):
13B INT4 with ~25% runtime headroom ≈ 7.6 GiB
70B INT4 with ~25% runtime headroom ≈ 40.7 GiB

Smallest practical GPU:
13B INT4: 16 GB GPU
70B INT4: 48 GB GPU

Is the existing 48 GB card good enough for the school to use?
Yes. The 48 GB card comfortably fits the 13B model. It can also run the 70B model at INT4, but that option leaves much less runtime headroom.

2. VECTOR DATABASE SIZING

The selected precision of vectors: halfvec

Reason:
For a deployment of the ministry size, halfvec reduces the storage requirements for vectors by about 50% compared to float32, but retrieves with almost the same quality.

Formula:
raw_vectors = num_vectors × dimensions × bytes_per_component
HNSW_overhead ≈ num_vectors × 8 × M bytes, with M = 32

Raw vectors:
5,000,000 × 1536 × 2 bytes
= 15,360,000,000 bytes
≈ 14.31 GiB

HNSW graph overhead:
5,000,000 × 8 × 32 bytes
= 1,280,000,000 bytes
≈ 1.19 GiB

Total vector estate:
14.31 GiB + 1.19 GiB = 15.50 GiB

5. TIERING TABLE

Asset: Model weights
Tier: hot
Reason: The reasons are: Live service of Reason, Latency is a concern.

Asset: Vector index
Tier: hot
Reason: Reads are latency critical; low latency reads are important;

Asset: RAG corpus
Tier: warm
Reason: Don't read frequently, but read on each token – throughput is more important than IOPS.

Asset: Fine-tuning dataset
Tier: warm
Reason: There is a major restriction on this important, but non-live, category.

Asset: Raw logs
Tier: cold
Reason: Anything that is left out and subject to audit and review is okay to read sequentially.

Asset: Scratch / temp embeddings
Tier: warm
Reason: temporary working set; writes faster, but only for the short term.

6. FINAL RECOMMENDATION

Recommended deployment:
13B model at INT4
halfvec vector storage
current 48 GB GPU retained

Reason:
The 13B option is not only convenient for the existing GPU, but also allows to maintain moderate storage space and does not necessitate any immediate upgrade of the storage device. The 70B option is viable on a 48 GB card on the INT4, but will leave the least headroom, and will be the "riskier" procurement option.
